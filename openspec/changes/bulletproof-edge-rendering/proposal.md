## Why

Despite the previous fixes ensuring that the inner fields (`data` and `next`) successfully generate, the structural pointer arrows for Linked Lists and Trees are still dropping from the canvas. 

The root cause is a **React Key Collision**. When the graph builder iterates over multiple stack variables (e.g., `a` and `b`), it redundantly parses the same connected structural links. As a result, it generates multiple identical edges (e.g., `edge-struct-0x1-next`) and pushes them into the `edges` array. When React Flow receives an array containing duplicate edge IDs, it triggers a catastrophic render collision, causing it to silently abort drawing the duplicated edges—leaving the nodes disconnected.

## What Changes

1. **Edge Deduplication**:
- We will transition the edge collection mechanism in `MemoryCanvas.tsx` from a simple `edges.push()` array to a `Map<string, Edge>`. 
- By using the edge's unique ID as the Map key, duplicate edges generated by overlapping structural paths will naturally overwrite each other. When parsing is complete, we convert the Map back into a clean, strictly unique array of edges, guaranteeing that React Flow renders every single arrow flawlessly without collisions.

2. **Immutable React Updates**:
- When overriding `existing.data.variables` with `subVariables` for structural nodes, we will ensure that we create a shallow clone of the `data` object (e.g., `ed.variables = [...subVariables]`) to guarantee that React's state diffing engine detects the deep mutation and forces the node to re-render its values and handles on screen.

## Capabilities

### Modified Capabilities
- `memory-visualization-ui`: Refactor the `edges` array into an `EdgeMap` algorithm to guarantee absolute referential uniqueness for React Flow.

## Impact
- **Frontend**: The structural connections (Trees, Linked Lists) will flawlessly connect with arrows because the render engine will no longer crash from duplicated edge keys. Variables will confidently re-render when overridden.
